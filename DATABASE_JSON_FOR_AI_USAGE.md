## AIが準備書面作成に必要な情報

### 準備書面に必要な要素

準備書面は以下の構成要素から成ります：

```
準備書面
├── 事件の表示（事件番号、当事者名）
├── 請求の趣旨
├── 請求の原因
│   ├── 事実関係の記述
│   │   ├── 時系列に沿った事実の展開
│   │   ├── 各事実を裏付ける証拠の引用
│   │   └── 証拠の内容説明
│   └── 法的主張
│       ├── 適用法令
│       ├── 法的構成
│       └── 相手方主張への反論
└── 証拠説明書
    ├── 証拠番号
    ├── 証拠の標目
    ├── 作成年月日
    └── 立証趣旨
```

### database.jsonがカバーする範囲

| 要素 | database.jsonの対応 | 該当フィールド |
|------|-------------------|--------------|
| **事件の表示** | ✅ 完全対応 | `case_info.*` |
| **事実関係の記述** | ✅ 完全対応 | `evidence[].phase1_complete_analysis.ai_analysis.objective_analysis.*` |
| **時系列整理** | ✅ 完全対応 | `evidence[].phase1_complete_analysis.ai_analysis.objective_analysis.temporal_information.*` |
| **証拠引用** | ✅ 完全対応 | `evidence[].evidence_number`, `complete_description` |
| **証拠内容説明** | ✅ 完全対応 | `evidence[].phase1_complete_analysis.ai_analysis.objective_analysis.complete_description` |
| **証拠説明書** | ✅ 完全対応 | 全メタデータ + `document_date` |
| **法的主張** | ⚠️ Phase 2で追加予定 | 現在は客観的事実のみ |
| **相手方主張への反論** | ⚠️ Phase 2で追加予定 | - |

## database.jsonの設計思想

### Phase 1（現在）: 客観的事実記録

```
【設計原則】
1. 完全に客観的・中立的
2. 観察可能な事実のみを記録
3. 法的評価を含まない
4. 原文参照不要な完全言語化（レベル4）
5. すべてプログラム解釈可能

【目的】
→ AIが客観的データベースとして利用できる
→ Phase 2で法的分析を加える際の信頼できる基盤
```

### なぜこの設計なのか？

#### ❌ 悪い例（Phase 1で法的評価を含む）
```json
{
  "ai_analysis": {
    "complete_description": "本契約書は被告の債務不履行を明確に示している",
    "legal_significance": "原告に有利な決定的証拠",
    "relevance_to_case": "被告の責任を立証できる"
  }
}
```
**問題点**:
- 客観性が失われる
- 原告側のバイアスが混入
- 裁判所・相手方に提示できない
- Phase 2で再分析が必要

#### ✅ 良い例（Phase 1の現在の設計）
```json
{
  "objective_analysis": {
    "complete_description": "本契約書は2021年8月15日付で、甲（山田太郎）と乙（田中花子）の間で締結された。第5条に「乙は同年10月20日までに代金1000万円を支払う」と記載されている。乙の署名欄には署名と印鑑があり、日付は2021年8月15日と記載されている。",
    "observable_facts": [
      "契約日: 2021年8月15日",
      "当事者: 甲（山田太郎）、乙（田中花子）",
      "支払期限: 2021年10月20日",
      "金額: 1000万円",
      "乙の署名・押印あり"
    ],
    "temporal_information": {
      "document_date": "2021-08-15",
      "document_date_source": "契約書末尾の契約締結日",
      "other_dates": [
        {"date": "2021-10-20", "context": "支払期限"}
      ]
    }
  }
}
```
**利点**:
- 完全に客観的
- どの立場でも利用可能
- 裁判所・相手方にも提示できる
- Phase 2で自由に法的分析を加えられる

## AIプロンプト例

### プロンプト1: 準備書面の「請求の原因」部分を作成

```
あなたは民事訴訟の準備書面を作成する弁護士です。

以下のdatabase.jsonを読み込み、時系列に沿って事実関係を整理し、
各事実を裏付ける証拠を引用しながら「請求の原因」部分を作成してください。

【database.json】
{database.jsonの内容を全文貼り付け}

【作成指示】
1. 時系列に沿って事実を記述
2. 各事実の後に「（甲○号証）」と証拠番号を引用
3. 重要な証拠内容は詳しく説明
4. 客観的な事実のみを記述（法的評価は別途記載）
5. 段落番号を付けて構造化

【出力形式】
第1 当事者
1. 原告は...
2. 被告は...

第2 事実関係
1. 契約の成立
  (1) 原告と被告は、令和3年8月15日、...（甲1号証）
  (2) 本契約書には...
...
```

### プロンプト2: 証拠説明書を作成

```
以下のdatabase.jsonから証拠説明書を作成してください。

【database.json】
{database.jsonの内容}

【出力形式】
証拠説明書

号証 | 標目 | 作成年月日 | 作成者 | 立証趣旨
-----|------|-----------|--------|----------
甲1  | 契約書 | R3.8.15 | 甲・乙 | 本件契約の成立及び内容
甲2  | 領収書 | R3.10.20 | 乙 | 代金の一部支払い
...

【データ取得方法】
- 号証: evidence[].evidence_number
- 標目: evidence[].original_filename から判断
- 作成年月日: evidence[].phase1_complete_analysis.ai_analysis.objective_analysis.temporal_information.document_date
- 作成者: evidence[].phase1_complete_analysis.ai_analysis.objective_analysis.parties_mentioned から判断
- 立証趣旨: evidence[].phase1_complete_analysis.ai_analysis.objective_analysis.complete_description から要約
```

### プロンプト3: 時系列表を作成

```
database.jsonから時系列表を作成してください。

【database.json】
{database.jsonの内容}

【出力形式】
日付       | 出来事 | 証拠 | 詳細
-----------|--------|------|------
R3.8.15    | 契約締結 | 甲1 | 原告と被告の間で...
R3.10.20   | 支払期限 | 甲1 | 契約書第5条に定める支払期限
R3.10.25   | 催告 | 甲2 | 原告が被告に対し...
...

【データ取得方法】
1. すべての証拠から temporal_information.document_date を取得
2. temporal_information.other_dates も含める
3. 日付順にソート
4. complete_description から出来事を要約
```

## フィールドの重要度（AI利用観点）

### 🔴 最重要（準備書面作成に必須）

| フィールド | 用途 | 準備書面での利用箇所 |
|-----------|------|-------------------|
| `evidence_number` | 証拠引用 | 「（甲1号証）」 |
| `objective_analysis.complete_description` | 証拠内容説明 | 事実関係の記述 |
| `temporal_information.document_date` | 時系列整理 | 事実関係の時系列記述、証拠説明書 |
| `temporal_information.document_date_source` | 日付の根拠 | 証拠の信頼性説明 |
| `observable_facts` | 事実の箇条書き | 事実関係の構造化 |
| `parties_mentioned` | 登場人物 | 当事者・関係者の整理 |
| `financial_information` | 金額情報 | 請求金額の根拠 |

### 🟡 重要（補助的に使用）

| フィールド | 用途 | 準備書面での利用箇所 |
|-----------|------|-------------------|
| `temporal_information.timeline` | 時系列の流れ | 事実関係の全体像把握 |
| `temporal_information.other_dates` | 関連日付 | 副次的な時系列情報 |
| `document_state` | 文書の状態 | 証拠の信頼性説明 |
| `case_info.*` | 事件情報 | 準備書面の冒頭部分 |
| `original_filename` | ファイル名 | 証拠説明書の「標目」 |

### 🟢 参考（必要に応じて使用）

| フィールド | 用途 | 準備書面での利用箇所 |
|-----------|------|-------------------|
| `hashes.*` | ファイル同一性証明 | 証拠の真正性主張 |
| `gdrive.file_url` | 証拠へのアクセス | （準備書面には記載しない） |
| `quality_assessment.*` | 分析品質確認 | （準備書面には記載しない） |
| `exif.*` | 撮影情報 | 写真証拠の真正性主張 |

### ⚪ 不要（準備書面作成には使わない）

| フィールド | 理由 |
|-----------|------|
| `file_size_bytes` | 準備書面に無関係 |
| `file_permissions` | 技術情報 |
| `extraction_info.*` | システム内部情報 |
| `analysis_version.*` | システム内部情報 |
| `processed_file_path` | ローカルパス（他環境で無効） |

### 現在の使い方（Phase 1のみで準備書面作成）

Phase 2が未実装の現在でも、以下のように使えます：

```
1. database.jsonをAI（ChatGPT, Claude等）に読み込ませる

2. プロンプトで指示:
   「原告の立場から、時系列に沿って事実関係を整理し、
    証拠を引用しながら請求の原因を作成してください」

3. AIが出力:
   - Phase 1の客観的事実を基に
   - 原告側の視点で解釈・整理
   - 準備書面形式で出力

4. 人間がレビュー・修正:
   - 法的主張の追加
   - 表現の調整
   - 相手方主張への反論追加
```

**メリット**:
- Phase 1の客観的データが信頼できる基盤になる
- AIが勝手な解釈をしにくい（事実ベースで記述される）
- 複数の法的構成を試しやすい

#### AIへのプロンプト

```
あなたは原告（株式会社山田商事）の訴訟代理人弁護士です。

以下のdatabase.jsonを読み込み、被告（田中太郎）に対する
売買代金請求訴訟の準備書面を作成してください。

【database.json】
{上記のJSON}

【作成指示】
1. 「請求の原因」部分を時系列に沿って記述
2. 各事実の後に証拠番号を引用（例:「（甲1号証）」）
3. 重要な証拠内容は詳しく説明
4. 段落番号を付けて構造化
5. 法的主張も含める（民法等の適用）

第2 請求の原因

#### AIの出力例

```
準備書面

令和○年（ワ）第○○号 売買代金請求事件
原告 株式会社山田商事
被告 田中太郎

第1 請求の趣旨
1 被告は原告に対し、金1000万円及びこれに対する令和3年10月15日から
  支払済みまで年3パーセントの割合による金員を支払え
2 訴訟費用は被告の負担とする
との判決並びに仮執行宣言を求める。

第2 請求の原因
1 契約の成立
 (1) 原告と被告は、令和3年8月15日、原告が被告に対し商品A100個を
    売り渡し、被告がこれを買い受ける旨の売買契約（以下「本件契約」
    という）を締結した（甲1号証）。

 (2) 本件契約書第5条には「買主は売主に対し、本契約締結日から60日
    以内（2021年10月14日まで）に代金1000万円を支払う」と明記されて
    いる（甲1号証）。

 (3) 本件契約書には原告代表者と被告の署名・押印があり、有効に契約が
    成立している（甲1号証）。

2 履行
 (1) 原告は、令和3年9月10日、被告に対し、本件契約に基づき商品A100個
    を納品した（甲2号証）。

 (2) 甲2号証の納品書には、納品先「田中太郎」、納品元「株式会社山田商事」、
    品名「商品A」、数量「100個」、納品日「2021年9月10日」と記載されて
    おり、原告の社印が押印されている。

3 債務不履行
 (1) 被告は、本件契約に基づき、令和3年10月14日までに代金1000万円を
    支払う義務を負っていた。

 (2) しかし、被告は同日を経過しても代金を支払わなかった。

 (3) 原告は、令和3年10月15日、被告に対し、本書到達後7日以内に代金
    1000万円を支払うよう催告した（甲3号証）。

 (4) 甲3号証の催告書には「貴殿は当社との間で令和3年8月15日に締結
    した売買契約に基づき、同年10月14日までに代金1000万円を支払う
    義務を負っているところ、未だ支払いがありません。つきましては、
